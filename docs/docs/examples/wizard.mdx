# Full-Screen Wizard

A **full-screen onboarding wizard** with a persistent progress bar in the chrome, async step loading, and a final `resolve` that returns the collected data.

Steps 2 and 3 load asynchronously (simulating code-split chunks). The progress bar updates **immediately** on `advance` — before the async step resolves — because chrome renders outside Suspense.

Click **Start Onboarding** below to try it.

```jsx live
function WizardDemo() {
  const ref = useRef(null);
  const { initFlow } = useFlowInit();
  const [log, setLog] = React.useState("");

  // --- Chrome ---
  function WizardChrome({ children }) {
    const ctx = useFlowContext();
    const pct = Math.round((ctx.step / ctx.total) * 100);
    return (
      <div style={{ position: "fixed", inset: 0, background: "#fff", display: "flex", flexDirection: "column", zIndex: 1000 }}>
        <div style={{ height: 4, background: "#eee" }}>
          <div style={{ height: "100%", background: "#4f46e5", width: `${pct}%`, transition: "width 0.3s ease" }} />
        </div>
        <div style={{ padding: "8px 24px", color: "#888", fontSize: 13, borderBottom: "1px solid #eee" }}>
          Step {ctx.step} of {ctx.total}
        </div>
        <div style={{ flex: 1, display: "flex", alignItems: "center", justifyContent: "center" }}>
          {children}
        </div>
      </div>
    );
  }

  // --- Steps ---
  function StepProfile() {
    const { advance, abort } = useStep();
    return (
      <div style={{ textAlign: "center" }}>
        <h2 style={{ margin: "0 0 8px" }}>Tell us about yourself</h2>
        <p style={{ margin: "0 0 16px", color: "#666" }}>What should we call you?</p>
        <div style={{ display: "flex", justifyContent: "center", gap: 8 }}>
          <button onClick={() => advance(loadStepPreferences, { step: 2, total: 3, data: { name: "Demo User" } })}>
            Continue →
          </button>
          <button onClick={() => abort("cancelled")} style={{ background: "none", color: "#888", border: "1px solid #ddd", borderRadius: 4, padding: "4px 12px", cursor: "pointer" }}>
            Cancel
          </button>
        </div>
      </div>
    );
  }

  function StepPreferences() {
    const { retreat, advance, abort } = useStep();
    const ctx = useFlowContext();
    return (
      <div style={{ textAlign: "center" }}>
        <h2 style={{ margin: "0 0 8px" }}>Set your preferences</h2>
        <p style={{ margin: "0 0 16px", color: "#666" }}>Enable notifications?</p>
        <div style={{ display: "flex", justifyContent: "center", gap: 8 }}>
          <button onClick={() => retreat()} style={{ background: "none", border: "1px solid #ddd", borderRadius: 4, padding: "4px 12px", cursor: "pointer" }}>← Back</button>
          <button onClick={() => advance(loadStepConfirm, { ...ctx, step: 3, data: { ...ctx.data, notifications: true } })}>
            Continue →
          </button>
          <button onClick={() => abort("cancelled")} style={{ background: "none", color: "#888", border: "1px solid #ddd", borderRadius: 4, padding: "4px 12px", cursor: "pointer" }}>
            Cancel
          </button>
        </div>
      </div>
    );
  }

  function StepConfirm() {
    const { retreat, resolve, abort } = useStep();
    const ctx = useFlowContext();
    return (
      <div style={{ textAlign: "center" }}>
        <h2 style={{ margin: "0 0 8px" }}>All set! 🎉</h2>
        <p style={{ margin: "0 0 16px", color: "#666" }}>Your profile is ready.</p>
        <div style={{ display: "flex", justifyContent: "center", gap: 8 }}>
          <button onClick={() => retreat()} style={{ background: "none", border: "1px solid #ddd", borderRadius: 4, padding: "4px 12px", cursor: "pointer" }}>← Back</button>
          <button onClick={() => resolve(ctx.data)}>Finish</button>
          <button onClick={() => abort("cancelled")} style={{ background: "none", color: "#888", border: "1px solid #ddd", borderRadius: 4, padding: "4px 12px", cursor: "pointer" }}>
            Cancel
          </button>
        </div>
      </div>
    );
  }

  // --- Async loaders ---
  const loadStepPreferences = () =>
    new Promise((res) => setTimeout(() => res({ default: StepPreferences }), 700));
  const loadStepConfirm = () =>
    new Promise((res) => setTimeout(() => res({ default: StepConfirm }), 700));

  // --- Host ---
  async function start() {
    setLog("");
    try {
      const result = await initFlow(StepProfile, ref, { step: 1, total: 3, data: {} });
      setLog(`Onboarding complete: ${JSON.stringify(result)}`);
    } catch {
      setLog("Onboarding cancelled.");
    }
  }

  return (
    <div style={{ padding: 8 }}>
      <button onClick={start}>Start Onboarding</button>
      {log && <p style={{ marginTop: 12, color: "#555" }}>{log}</p>}
      <FlowOutlet
        ref={ref}
        fallback={<span style={{ color: "#aaa" }}>Loading…</span>}
        chrome={(slot) => <WizardChrome>{slot}</WizardChrome>}
      />
    </div>
  );
}
```

## Async step loading

Steps 2 and 3 use async loaders — functions that return `Promise<{ default: ComponentType }>`, mirroring a real dynamic `import()`:

```js
// Real-world code splitting
const loadStepPreferences = () => import("./StepPreferences");

// Simulated in this example
const loadStepPreferences = () =>
  new Promise((res) => setTimeout(() => res({ default: StepPreferences }), 700));
```

`advance(loader, newContext)` accepts either a component directly or an async loader. While the loader is pending, the `<Suspense>` fallback inside `WizardChrome` shows the loading indicator.

## Progress via context

The progress bar reads `ctx.step` and `ctx.total` from `useFlowContext()`. Each `advance` call passes updated context:

```js
advance(loadStepPreferences, { step: 2, total: 3, data: { name: "…" } })
```

Because chrome renders **outside** the Suspense boundary, the progress bar jumps from 33% → 66% **immediately** when `advance` is called — before the next step finishes loading.

## Key points

- **Chrome outlives async loads** — the progress bar and step counter stay mounted while the next step resolves.
- **Context carries accumulated data** — each step merges its data into context so the final step can `resolve(ctx.data)`.
- **No centralized step list** — the total step count is just data passed through context; you can change it without touching any config.
